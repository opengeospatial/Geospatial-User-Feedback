[[abstract]]
[big]*i.     Abstract*
{set:bgcolor:#D5F5E3}

The OGC API standards family includes a set of resources that enable anyone to provide or use geospatial information. These APIs primarily describe the publication and access to data (e.g. features), although they may also reference metadata (i.e. records).

This Standard takes a step further by defining the OGC API - Geospatial User Feedback (GUF), which builds upon the concept of OGC API - Records. OGC API - Records is designed to catalogue records. Commonly these records contain metadata associated with resources created by the producer and created from the producer's point of view. Instead, OGC API - GUF specifies how to provide user feedback on any catalog entry, whether a geospatial collection or a metadata record in a OGC API Records collection. This mechanism focuses on the users perspective, allowing them to comment, rate, describe their usage, report errors or deficiencies they have found, link the data or metadata to related documentation, or report on bits of knowledge they have produced and want to share, etc. Moreover, it enables the reporting of statistics on these evaluations, which can inform other users.

Thus, in OGC API - Geospatial User Feedback, a feedback item is the atomic unit of information within a collection, just as a record is in OGC API - Records. User Feedback Items must be accessed conforming to this Standard.

Accordingly, this Standard defines the following set of components for OGC API - Geospatial User Feedback:
* a feedback item
* a feedback collection (that acts as a feedback catalog)
* a feedback statistic
* a common set of query parameters
* the requirements and characteristics for implementing a user feedback API

Since the very nature of feedback is based on user contributions, OGC API - GUF allows not only the HTTP GET method for queries, but also adopts from OGC API - Features - Part 4 the capabilities for creating, modifying and deleting feedback items using HTTP POST, PUT and DELETE methods.

//== Security considerations

See OGC API - Features - Part 1: Core, Clause 11.

See OGC API - Features - Part 4: Create, Replace, Update and Delete, Clause 11.

In addition to what is defined in OGC API – Features – Part 1 and OGC API – Features – Part 4, a common OGC API – Geospatial User Feedback service expects contributions from many different users, which makes privilege management different from that of a typical OGC API – Features implementation. These are the common rules for managing feedback:

Users making modifications to resources shall comply with the following:
* Any user who is able to authenticate has the privilege to create new feedback.
* Authenticated users are authorized to modify or delete only the records they have created.
* Some special users, acting as moderators, have the privilege to modify or delete records created by any user.

The API definition shall reflect these rules in the resource paths and their available methods.

The service provider should make every reasonable effort to ensure that feedback records are trustworthy and do not contain false, harmful, or irrelevant information. Active moderators (human or AI) should review the records and remove any that have been proven to be incorrect. Users who repeatedly include incorrect information should have their privileges revoked, their authentication rights deactivated, and, eventually, all their contributed feedback that cannot be proven correct removed.
